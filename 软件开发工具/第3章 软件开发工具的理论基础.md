## 3.1 软件开发过程的信息需求

1. 软件开发过程中，主要有几方面的信息

   - 1）有关系统环境、现状及`需求的信息`。
   - 2）有关`软件的功能设计与物理设计`的各种信息。
   - 3）软件成果本身，包括`程序与文档`。
   - 4）用户对系统的各种变更要求，以及系统的各种变更的记录。事实上，这类信息是对前三类信息的更新、修改或补充。

2. 围绕以上四类信息，涉及的信息管理工作

   - 1）许多信息需要保存，不断更新、不断被调用
   - 2）在许多环节上都要进行数据的转换或加工。最突出的是各种人机界面。

3. 相关的理论与技术

   - 1）认知科学中关于`概念模式`的概念与方法

   - 4）关于`人机界面`的理论与方法
   - 5）管理科学中关于`项目管理与版本管理`的理论与方法
   - 6）系统科学与系统工程中的有关理论与方法

## 3.2 概念模式及其作用

1. 概念模式

   - 它是指人们在认识事物过程中，对于某一事物或某一系统形成的、抽象的、一般化的框架。
   - 概念模式是人们认识客观世界的一种方法、一种工具。

2. 在软件开发中常用的 7 种概念模式

   - 1.框图

     - 它是用来描述程序执行的逻辑过程。它把程序的基本步骤归纳为处理、判断、输入输出、起始或终结等几个基本功能，并用不同的记号加以表示。

   - 2.结构图

     - 用以表示大型软件的层次结构，即模块结构。它的基本观点是，程序的调用方式有三种：顺序调用、选择调用、循环调用。

   - 3.数据流程图

     - 它的基本思想是把信息流看做一个组织或系统运作的线索，力图简明扼要地勾画出全局。

   - 4.实体关系图（E-R 图）

     - 是一种用于描述静态数据结构的概念模式，它以实体、关系、属性三个基本概念概况数据的基本结构。

   - 5.数据字典（Data Dictionary）

     - 也是一种描述数据内容的概念模式。它用表格的形式列出数据的基本属性以及相互关系，它的雏形是编写软件时的变量豁免或标识符清单。

   - 6.时序网络
   - 7.数学与逻辑模型
   - 8.计算机模拟模型

## 3.3 信息库及其一致性

1. 信息传递中的两个问题

   - 第一个问题，在信息传递中，误解与丢失十分严重。
   - 第二个问题，是由软件的修改与版本更新引起的。

2. 信息库

   - 信息库是一个包罗万象的，随着项目进度不断修改与补充的数据集合。它的数据结构相当复杂，而且会随着时间不断变化，因此保持一致性的任务变得十分复杂和艰巨。

3. 信息库的研究

   - 1）信息库的内容应当包含哪些方面
     - 笼统的说，信息库中应当存放有关软件全过程的所有有用的信息
   - 2）**信息库应当具备哪些管理功能**

     | 阶段         | 信息                                                                                                               |
     | :----------- | ------------------------------------------------------------------------------------------------------------------ |
     | 外部信息     | 软件的工作环境、功能需求、性能要求、有关的各种信息来源的状况等等                                                   |
     | 需求分析阶段 | 收集的有关用户的各种信息                                                                                           |
     | 逻辑涉及阶段 | 各种调查材料和由此生成的各种文档，这包括调查记录、原始数据、报表及单证的样本、绘制的各种图以及最后生成的系统说明书 |
     | 设计阶段     | 包括所有的数据库与数据文件格式、数据字典、程序模块的要求、总体结构、各种接口及参数的传递方式及最后形成的设计方案   |
     | 编程阶段     | 程序代码、框图、变量说明、测试情况、验收报告、使用说明                                                             |
     | 运行及使用   | 运行及使用情况的详细记录                                                                                           |
     | 维护及修改   | 维护及修改的情况                                                                                                   |
     | 项目管理     | 项目管理的有关信息，包括人员变更、资金投入、进度计划及实施情况，还包括版本信息                                     |

   - 3）如何保持一致性

## 3.4 人机界面及其管理

- ![image](https://img2018.cnblogs.com/blog/704904/201910/704904-20191010224009058-1264735790.png)

1. 人机界面的重要性

   - 对于用户，界面就是系统本身。
   - 由于计算机技术的飞速发展，计算机的速度与内存、外存都已经不再是软件开发人员所担心的问题。对于用户来说，所关心的最主要问题本来就是自己是否能否有效地使用软件，而不是系统内部的算法或数据结构。当然，这并不是说这些问题不重要，它们终究使计算机各种功能得以实现的基础。人们已经公认，在当今的硬件与软件的环境下，如果一个软件没有很好的界面设计，就不能算是成功。

2. 关于用户界面的基本原则

   - 1）用户界面的主要功能是通信

     - 用户界面的主要功能使通信，关键要使人与计算机之间能够准确地交流信息。一方面，在由人向机器输入时，应当尽量采取自然的方式；另一方面，由机器向人传递的信息必须准确、不致引起误解或混乱，用词必须准确。另外，不要把内部的处理、加工与人机界面混在一起，人机界面的主要功能只是通信，以免互相干扰及影响速度。

   - 2）用户界面必须始终一致

     - 用户界面必须始终一致。决不要增加用户的负担，让用户可以始终用同一种方式思考与操作。最忌讳的是换一个屏幕用户就要换一套操作命令与操作方法。当然这种一致性是要根据用户的习惯与要求。

   - 3）用户界面必须使用户随时掌握任务的进展状况

     - 用户使用中有时会分心去处理别的事情，当它回到终端前时，人机界面应该能告诉他刚才进行到什么地方。

   - 4）用户界面必须能提供帮助

     - 用户界面必须能提供帮助。决不可以认为使用程序的都是专家，无须提供帮助。有关的信息、提示、说明应该放在随手可得的位置。

   - 5）宁可让程序多干，不可让用户多干

     - 所有这些原则当然是要花费代价的，如联机帮助系统常常要形成一个相当大的模块，对各种错误操作的预防也需要付出相当多的劳动，但是所有这些代价都是值得的，让用户使用方便会得到更大的回报。

   - 人机界面设计中经常遇到的一个困难的权衡：灵活性和一致性的矛盾。原则上，对于某一软件来说，用户的思维方式应当是一致的，操作与控制的方式应当是一致的。只有这样，才能减轻用户使用的困难，使用户不知不觉地进入统一的思维框架，从而顺利地、正确地应用这个软件。另一个方面的理由是，用户是多种多样的，她们有各自的习惯与爱好，应当让他们尽可能灵活、自由地选择某些功能的操作方式。

3. 人机交互手段
   - 键盘操作
   - 屏幕滚动
   - 菜单选择
   - 帮助系统
   - 鼠标操作
   - 色彩应用
   - 数据录入
   - 信息显示

## 3.5 项目管理与版本管理

1. 项目管理

   - 含义：是指与固定的生产线上的日常生产管理不同的，具有更大的变动性、时间性的一类管理任务。
   - 它的典型代表是建筑工程的管理，特别是大型建设工程，如水电站的建造、高炉的大修或改造、铁路或公路的修建等。

2. 项目管理的四个突出特点

   - 1）子任务多，关系复杂

     - 这类工程项目常常包括成百上千个子任务，它们之间既有前后工序的制约关系，又有许多并行作业。与一般的、固定的生产线相比，情况更为复杂。

   - 2）任务不可重复，形势不断变更

     - 上述这些子任务一旦完成，至少在本项目期间就不再做了，不存在“下一次”。

   - 3）协调组织的任务十分突出，资源浪费闲置的风险与合理地优化组合、提高效益的机会并存

     - 如果组织不当，不仅会造成大量人力、物力、设备的浪费，而且会严重地影响工程的进度与质量，而合理安排可能带来的效益也是非常可观的。

   - 4）信息处理工作的作用与意义更为突出

     - 上面所说的问题，即最终效果如何，取决于正确的指挥决策，而决策的正确则立足于这种管理工作，信息分散，难以采集也是普遍的现象。

   - `进度和质量`是软件开发最关键的问题

3. 项目管理的基本目标

   - 1）是产品或工程的质量得到有效的控制
   - 2）保证整个系统按预定的进度完成
   - 3）有效地利用各种资源，尽可能使资源的闲置与浪费减少；（最重要的资源：程序员的时间和能力）
   - 4）控制与降低成本。（成本的情况使最终体现）
   - 为了达到这些要求，人们采用工程计划网络方法、甘特图、检查点方法、排队论等。

4. 版本管理的核心

   - 版本管理的核心使保持两个一致性。
     - 时间上关心系统的逐步完善，以前成果不要丢失，可以回溯、重用。而每一步新的进步与以前工作一致。
     - 从系统各部分之间的关系来说，要求每一局部与整体保持一致，合理地判断每一次修改的必要性及影响范围。

5. 版本管理的方法
   - 具体方法包括规定长远的版本更新计划，制定与版本有关的信息范围及收集、管理方法，在项目组内明确分工，在程序编写、测试、文档缩写等工作中贯穿项目管理的思想及要求。
   - 无论对于软件产品来说，还是对应用系统来说，用户的反馈信息使版本管理的重要资源。

3. 什么叫做概念模式？它的作用和意义是什么？

   - 作用和意义
     - 人们在互相配合、互相协调、共同完成某项任务的时候，需要统一思想、统一认识、协同行动就必须用某种公认的方式来进行表示，概念模式就是起了这种作用。
     - 概念模式的意义决定了它的相对性，即面对各种不同的用途，必然会有不同形式的概念模式

4. 举出三种常用的概念模式，说明它们的作用与使用方法？

   - 1）框图。作用：用来描述程序执行的逻辑过程。使用方法：把程序的基本步骤归纳为处理、判断、输入输出、起始或终结等几个基本功能，并用不同的记号加以表示。
   - 2）结构图。作用：用来表示大型软件的层次结构。使用方法：基本图例是模块，用自上而下的连线表示调用关系，并注明参数传递的方向与内容。它的基本观点是，程序的调用方式有三种：顺序调用、选择调用、循环调用。它用箭头上的附加记号来表明这一点。
   - 3）数据流程图。作用：用来描述某一业务处理系统的信息来源、存储、处理、去向的全面情况。使用方法：数据流程图的基本元素是外部实体、数据处理与数据存储。用箭头表示信息在它们之间的流动状况。

5. 什么叫信息库？它与数据库、数据仓库有什么区别？

   - 数据库是在文件系统基础上发展起来的一种数据管理技术，它的重点是在于合理地组织数据，体现出数据之间的逻辑关系，并尽可能地帮助用户检索，减少冗余。
   - 数据仓库是近年来针对大批数据的分析利用而提出来的概念。它的基本思想是：现实中的大量原始数据与实际的应用是有相当距离的，为了能够切实为企业管理者提出对管理真正有用的信息，就要对信息或数据进行多次的加工或提炼，从而得出许多派生的综合的数据。
   - 信息库是一个包罗万象的，随着项目进度不断修改与补充的数据集合。
   - 信息库在规模上不一定像数据仓库那么大，分析提炼的要求也与数据仓库不同，但是它的数据结构相当复杂。它是针对软件开发或信息系统开发中的大量信息管理工作提出来的。

6. 为什么信息库的一致性十分重要？

   - 因为在信息传递和软件的修改与版本更新中，误解与丢失十分严重。

9. 什么叫做项目管理？其任务是什么？

   - 项目管理是指与固定的生产线上的日常生产管理不同的、具有更大的变动性、时间性的另一类管理任务。
   - 项目管理的任务
     - 1.使产品的质量得到有效的控制。
     - 2.保证整个系统按预定的进度完成。
     - 3.有效地利用各种资源，尽可能使资源的闲置与浪费减少。
     - 4.控制与降低成本。

10. 什么叫做版本管理？其任务是什么？
    - 版本管理：在软件产品经过若干次改进与更新的过程中要求项目组保持项目的持续性，使以往的信息能够完整地保存，并能够在需要时被取出使用。
    - 任务：保持两个一致性
      - 从时间上来说，它的主要关心点是系统的逐步完善，以前已经达到的成果不要丢失掉。而每一步新的进步都要切实与以前的工作一致，是改进而不是破坏或降低系统的已有功能。
      - 从系统各部分之间的关系来说，它要求每一局部与整体保存一致，合理地判断每一次修改的必要性及影响范围，使新的版本成为系统进步的漫长过程中的一个进步。

      （4）用户对系统的各种变更要求，以及系统的各种变更的记录。

11.【2016.04】信息库中应保存哪些关于维护与修改的内容？

    - 包括修改的目标、责任、过程、时间、修改前后的代码，文档以及修改后的结果、原系统的备份。

12.【2020.10】结合你的实际体会谈谈用户界面设计的原则。（10 分）

    - （1）用户界面的主要功能是通信；
      （2）用户界面始终一致；
      （3）使用户掌握任务进展情况；
      （4）给用户提供帮助；
      （5）宁可让程序多干，不可让用户多干。

13. 【2019.10】与一般工厂的日常生产管理相比，项目管理有什么特点？它的基本目标是什么？

    - 项目管理是指与固定生产线上的日常生产管理不同的，有更大变动性、时间性的一类管理。
    - 特点：子任务多，关系复杂；任务不可重复；协调组织任务繁重；信息处理工作更突出。
    - 基本目标：质量控制，保证进度，有效利用资源，控制成本。

14.【2019.04】什么是概念模式？它有什么作用？

    - （1）概念模式是指人们在认识事物时，对某事物或系统形成抽象的、一般化的框架。
      （2）概念模式具有客观性。
      （3）又有主观性。
      作用：
      （1）它是人们认识客观世界的一种工具。
      （2）是人们交流与表达的工具。

15.【2016.10】试述数据流程图的组成和作用。

    - 数据流程图由外部实体、数据处理、数据存储和数据流组成。数据流程图面对的是一个系统的信息流程。用于描述某一业务处理系统的信息来源、存储、处理、去向的全面情况。其基本思想是把信息流看作一个组织或系统运作的线索，简明扼要地描述处理的过程。数据流程图不仅应用于描述已有系统的状况，也应用于描述设想中新系统的状况。

16.【2016.04】试述软件项目管理的特点和基本目标。

      - 特点：
      （1）子任务多，关系复杂；
      （2）任务不可重复，形势不断更变；
      （3）协调组织的任务突出；
      （4）资源浪费风险与提高效益的机会并存；
      （5）信息处理工作的作用与意义突出。
      基本目标：
      （1）有效地控制产品（或工程）的质量；
      （2）保证整个系统按预定的进度完成；
      （3）有效地利用资源，尽可能使资源的闲置和浪费减少；
      （4）控制与降低成本。
