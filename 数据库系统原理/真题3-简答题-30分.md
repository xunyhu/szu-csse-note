     （4）特定的应用程序是在外模式的逻辑结构上编写的，它依赖于特定的外模式，与数据库的模式和存储结构独立。

5. 【2019.10】简述数据模型的分类。

   - 数据库系统的模式如若发生改变，数据库管理员（DBA）通常会对各个外模式/模式的映像做出相应的改变，以使那些对用户可见的外模式保持不变，从而应用程序的编程人员就不必去修改那些依据数据的外模式所编写的应用程序，如此实现了外模式不受概念模式变化的影响，并保证了数据与程序的逻辑独立性。

## 第 2 章 关系数据库

1. 【2023.04】简述关系模型的三类完整性约束

   答：实体完整性约束；参照完整性约束；用户定义完整性约束。

2. 【2022.10】简述关系模型完整性约束的检验方法

   答：1）执行插入操作时，首选检查实体完整性约束，然后检查参照完整性约束，最后检查用户定义完整性约束。  
    2）执行删除操作时，一般只需要对被参照关系检查参照完整性约束。  
    3）执行更新操作时，因为更新操作可看成时先执行删除操作，再执行插入操作，因此是上述两种情况的综合。

3. 【2022.04】简述关系的两个不变性及其含义

   答：1）实体完整性和参照完整性是关系模型必须满足的完整性约束条件，被称为`关系的两个不变性`。  
    2）`实体完整性约束是指关系的主属性`，即主码的组成不能为空，也就是关系的主属性不能是空值 NULL  
    3）`参照完整性约束是指定义外码和主码之间的引用规则`，它是对关系间引用数据的一种限制

4. 【2021.10】简述关系的码和候选码的概念

   答：1）如果在一个关系中存在这样的属性或属性组，使得在该关系中的任何一个关系状态中这些`属性或属性组的值能用来唯一标识该关系的元组`，则称为关系的码。  
    2）如果`在关系的一个码中移去任何一个属性它就不是这个关系的码`，则称这个码为该关系的候选码。

5. 【2021.04】简述关系数据库对关系限定的具体要求。

   （1）每个属性都是不可分解的。  
   （2）每一个关系仅仅有一种关系模式。  
   （3）每一个关系模式的属性必须命名，属性名不许重复。  
   （4）同一个关系中不允许出现候选码或候选键值完全相同的元组。
   （5）在关系中元组的顺序（即行序）是无关紧要的。  
   （6）在关系中属性的顺序（即列序）是无关紧要的。

6. 【2020.10】简述“不好”的关系模式可能存在的问题

   数据冗余；更新异常；插入异常；删除异常；

7. 【2020.08】简述关系模型完整性约束的检验

   答：1）执行插入操作时，首选检查实体完整性约束，然后检查参照完整性约束，最后检查用户定义完整性约束。  
   2）执行删除操作时，一般只需要对被参照关系检查参照完整性约束。  
   3）执行更新操作时，因为更新操作可看成时先执行删除操作，再执行插入操作，因此是上述两种情况的综合。

8. 【2019.10】简述实体完整性约束。

   - （1）实体完整性约束是指关系的主属性，即主码的组成不能为空，也就是关系的主属性不能是空值 NULL。  
     （2）关系对应于现实世界中的实体集，而现实世界中的实体是可区分的，即说明每个实体具有唯一性标识。  
     （3）在关系模型中，是使用主码作为唯一标识的，若假设主码取空值，则说明这个实体不可标识，即不可区分。

9. 【2019.04】简述参照完整性约束

   答：若属性 F 是基本关系 R 的外码，它与基本关系 S 的主码 K 相对应，则对于 R 中每个元组在 F 上的值只允许两种可能，即要么取空值，要
   么等于 S 中某个元组的主码值。

10. 【2018.10】简述用户定义完整性约束。

    - `用户定义的完整性约束是针对某一应用环境的完整性约束条件`，它反映了某一具体应用所涉及的数据应满足的要求。关系模型提供定义和检验这类完整性规则的机制，其目的是用统一的方式由系统来处理它们，不再由应用程序来完成这项工作。在实际系统中，这类完整性规则一般在建立数据库表的同时进行定义，也可以由应用编程人员在各模块的具体编程中通过程序进行检查和控制。

## 第 5 章 数据库编程

1. 【2023.04】简述使用存储过程的优点

   答：1）增强 SQL 语言的功能和灵活性；

2. 【2022.10】简述删除存储函数的语句及注意事项

   答：1）删除语句`drop function`

3. 【2022.04】简述存储过程与存储函数的区别

    2）存储过程要用 CALL 语句调用，存储函数直接调用  
    3）存储函数必须包含 RETRUN 语句，存储过程不允许包含该语句

4. 【2021.10】简述存储过程中常用的条件判断语句和循环语句

   答：1）条件判断语句`if...then...else语句和case语句`  
    2）循环语句`while语句、repeat语句和loop语句`

## 第 6 章 数据库安全与保护

1. 【2023.04】简述并发操作带来数据不一致的主要原因及解决办法

   答：1）主要原因是因为并发操作破坏了事务的隔离性；  
    2）解决办法有`封锁、时间戳、乐观控制法和多版本并发控制`；

2. 【2023.04】简述 MySQL 中备份数据的语句及其功能

   答：1）`select...info...outfile`  
    2）通过 select 语句将表`所有数据行`写入到一个`文件`中。

3. 【2022.10】简述三种典型的并发操作问题

   答：1）丢失更新。假设有两个事务 T1 和 T2，当它们同时读入同一个数据并加以修改时，事务 T2 的提交结果会破坏事务 T1 提交的结果。  
    2）不可重复读。两个事务 T1 和 T2，不可重复读是指事务 T1 读取数据后，事务 T2 执行更新操作，使事务 T1 无法再现前一次读取的结果。  
    3）读脏数据。两个事务 T1 和 T2，事务 T1 修改某一个数据，并将其写回磁盘，事务 T2 读取同一个数据后，事务 T1 由于某种原因被撤销，这时事务 T1 已经修改过的数据恢复原值，事务 T2 读到的数据就与数据库中的数据不一致，则事务 T2 读到的数据就为“脏”数据。

4. 【2022.04】简述三种典型的并发操作问题。

   答：1）丢失更新；2）读脏数据；3）不可重复读；

5. 【2022.04】简述 MySQL 中恢复数据的语句及数据恢复完毕之后对表解锁的语句

   答：1）恢复数据的语句是`LOAD DATA...INFILE`  
    2）解锁语句`UNLOCK TABLES`

6. 【2021.10】简述约束命名后，使用 ALTER TABLE 语句和 DROP TABLE 语句应该注意的问题

   答：1）完整性约束不能直接被修改。若要修改某个约束，实际是用 alter table 语句先刷除该约束，然后在增加一个与该约束同名的新约束。  
    2）使用 alter table 语句可以独立地删除完整性约束，而不会删除表本身  
    3）使用 drop table 语句删除一个表，则表中所有的完整性约束都会自动被删除

7. 【2021.10】简述数据库备份和数据恢复的含义

   答：1）数据备份是指通过`导出数据或复制表文件`的方式来制作`数据库副本`。  
    2）数据库恢复是当数据库`出现故障或遭到破坏时，将备份的数据加载到系统`，从而使数据库从错误状态恢复到备份时的状态。

8. 【2021.04】简述完整性约束条件的作用对象及其含义。

   作用对象可以是列、元组和表。  
   （1）列级约束，是指对列的类型、取值范围、精度等的约束。  
   （2）元组约束，是指元组中各个字段之间的相互约束。  
   （3）表级约束，是指若干元组之间、关系之间的联系的约束。

9. 【2020.10】简述封锁的工作原理。

   答：1）若事务 T 对数据 D 加了 X 锁，则所有别的事务对数据 D 的锁请求都必须等待直到事务 T 释放锁。  
    2）若事务 T 对数据 D 加了 S 锁，则别的事务还可以对数据 D 请求 S 锁，而对数据 D 的 X 锁请求必须等待直到事务 T 释放锁。  
    3）对读请求 S 锁，对更新（插入、删除、修改）请求 X 锁。  
    4）事务一直占有获得的锁直到结束（COMMIT 或 ROLLBACK）时释放。

10. 【2020.08】简述封锁的概念及封锁的类型。

    答：1）封锁是最常用的并发控制技术，基本思想是：需要时，事务通过向系统请求对它所希望的数据对象加锁，以确保它不被非预期改变。  
     2）排他锁（或 X 锁）和共享锁（或 S 锁）

11. 【2019.10】简述触发器的概念及其作用

    答：1）触发器是用户定义在关系表上的一类由事件驱动的数据库对象，也是一种保证数据完整性的方法。  
     2）主要作用是实现主键和外键不能保证的复杂的参照完整性和数据的一致性，从而有效地保护表中的数据。

12. 【2019.04】简述封锁的级别的内容

    答：1）0 级封锁，封锁的事务`不重复写`其他`非 0 级`封锁事务的`未提交的`更新数据  
     2）1 级封锁，被封锁的事务`不允许重写未提交`的更新数据  
     3）2 级封锁，被封锁的事务`既不重写也不读未提交`的更新数据  
     4）3 级封锁，被封锁的事务`不读未提交的更新数据，不写任何未提交数据`

13. 【2018.10】简述事务的概念及事务的特征

    答：1）事务是用户定义的一个数据操作序列，是一个完整的工作单元要么全部执行要么全部不执行，是一个不可分割的工作单位。  
     2）事务的特征:原子性(或 Atomicity)、一致性(或 Consistency)、隔离性(或 Isolation)和持续性(或 Durability)。
